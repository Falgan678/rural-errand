# 验证码功能调试指南

## 问题修复说明

### 修复的问题
1. ✅ **验证码没有过期时间** - 现在验证码有效期为10分钟
2. ✅ **登录和注册验证码互相覆盖** - 现在使用不同的key存储（`手机号_类型`）
3. ✅ **验证码可以重复使用** - 现在验证码使用后会被标记为已使用
4. ✅ **验证码验证逻辑不完善** - 现在有完整的验证流程

### 新的验证码机制

#### 1. 验证码存储结构
```python
sms_codes = {
    "17811619201_register": {
        "code": "123456",
        "expire_time": 1707456300.123,  # 时间戳
        "used": False
    },
    "17811619201_login": {
        "code": "654321",
        "expire_time": 1707456400.456,
        "used": False
    }
}
```

#### 2. 验证码验证流程
1. 检查验证码是否存在
2. 检查验证码是否已使用
3. 检查验证码是否过期
4. 验证验证码是否正确
5. 标记验证码为已使用
6. 5分钟后自动删除验证码

#### 3. 验证码有效期
- **发送后有效期**: 10分钟
- **使用后保留时间**: 5分钟（防止重复使用）

## 常见问题排查

### 问题1: 提示"验证码错误或已过期"

**可能原因：**
1. 输入的验证码不正确
2. 验证码已过期（超过10分钟）
3. 在登录页面获取验证码，却在注册页面使用（或反之）
4. 验证码已被使用过

**排查步骤：**
1. 访问 `http://localhost:8000/api/debug/sms-codes` 查看当前验证码
2. 检查服务器控制台日志，确认最新的验证码
3. 确认在正确的页面（登录/注册）使用对应的验证码
4. 检查验证码是否已过期或已使用

### 问题2: 验证码已使用

**原因：**
验证码只能使用一次，使用后会被标记为已使用。

**解决方法：**
重新获取新的验证码。

### 问题3: 验证码已过期

**原因：**
验证码有效期为10分钟，超时后自动失效。

**解决方法：**
重新获取新的验证码。

### 问题4: 登录和注册验证码混淆

**原因：**
登录和注册使用不同的验证码，不能混用。

**解决方法：**
- 在注册页面获取的验证码只能用于注册
- 在登录页面获取的验证码只能用于登录

## 生产环境建议

### 1. 使用Redis存储验证码
当前使用内存字典存储，重启服务会丢失。生产环境建议使用Redis：

```python
import redis

redis_client = redis.Redis(host='localhost', port=6379, db=0)

# 存储验证码
redis_client.setex(f"sms:{phone}:{type}", 600, code)

# 获取验证码
code = redis_client.get(f"sms:{phone}:{type}")
```

### 2. 添加发送频率限制
防止恶意刷验证码：

```python
# 同一手机号1分钟内只能发送1次
# 同一IP地址1分钟内只能发送3次
# 同一手机号1小时内最多发送5次
```

### 3. 添加图形验证码
在发送短信验证码前，先验证图形验证码，防止机器人攻击。

### 4. 配置真实短信服务
参考 `SMS_CONFIG.md` 配置腾讯云短信服务。

### 5. 关闭调试接口
生产环境应该关闭 `/api/debug/sms-codes` 接口，防止验证码泄露。
