## 0. 一句话产品介绍

**"乡邻快帮"是一款让村里人之间互相帮忙跑腿的微信小程序——你出钱发个需求（如代购、捎带、取件），同村或邻村的人接单赚点零花钱，比快递便宜，比拜托熟人不欠人情。**

## 1. 产品背景与目标用户

### 1.1 为什么做这个产品（背景）

| 痛点 | 现状 | 我们的解法 |
|---|---|---|
| 农村快递难 | 快递只到镇上，村民取件来回 5-20 公里 | 镇上的人顺路捎带，赚 3-10 元 |
| 老人腿脚不便 | 子女不在身边，买药、买菜要走远路 | 邻居帮跑一趟，付点辛苦费 |
| 农村闲置劳动力 | 留守妇女、退休老人有空闲时间 | 接附近的小单，每天赚 30-100 元 |
| 城里跑腿太贵 | 美团跑腿在县城起步价 8-15 元，农村还不接单 | 平台不抽成或低抽成（0-5%），价格由邻里自定 |

### 1.2 目标用户画像

**用户 A：发布者（"我有需求"）**
- 年龄：30-65 岁
- 身份：留守老人、宝妈、上班族（在外工作的子女帮老家下单）
- 场景：取镇上快递、买药、买菜、捎带、跑腿办事
- 痛点：自己没空/没车/没力气，又不好意思一直麻烦邻居白帮忙

**用户 B：跑腿员（"我来帮忙"，文档中叫"帮帮员"）**
- 年龄：18-55 岁
- 身份：村里有车/电动车的人、镇上学生、外卖员顺路、农闲劳动力
- 场景：去镇上办事顺路、专门接单赚外快
- 痛点：附近没有靠谱的兼职、闲着没事做

> **关键词：一人两角**——同一个用户既能当发布者，也能当跑腿员，不用注册两个账号。

### 2.2 订单状态机（最重要的一张图）

```
    [发布者下单]
         │
         ▼
   ┌──────────┐  跑腿员接单   ┌──────────┐ 跑腿员点送达 ┌──────────┐ 发布者评价 ┌──────────┐
   │ pending  │ ───────────▶ │ ongoing  │ ──────────▶ │completed │ ────────▶ │ reviewed │
   │ 待接单    │              │ 进行中    │             │ 已完成    │           │ 已评价    │
   └────┬─────┘              └────┬─────┘             └──────────┘           └──────────┘
        │                         │
        │ 发布者取消                │ 发布者/跑腿员 取消（需说明原因）
        ▼                         ▼
   ┌──────────────────────────────────────┐
   │           cancelled  已取消             │
   └──────────────────────────────────────┘
```

**核心规则**
- 只有 `pending` 和 `ongoing` 可以取消，`completed` 不可取消
- 评价只能由发布者发起，且订单必须是 `completed`
- 订单完成自动给跑腿员加收益，自动更新平均评分和好评率

## 3. 核心功能详细说明

> 以下每页都按"页面元素 → 输入/按钮 → 交互逻辑"三段式描写，开发可直接照着写代码。

### 3.1 P1 登录注册页

#### 页面元素
- 顶部 Logo + 文案"乡邻快帮 · 邻里互助跑腿"
- 三个 Tab 切换：`账号登录` / `验证码登录` / `注册`
- 测试账号快捷区（仅开发环境显示）：`📱 13800138001 / 123456` 点击可一键填充
- 底部"用户协议 + 隐私政策"勾选框（默认未勾选）

#### 输入框 & 按钮

| 字段 | 类型 | 校验规则 | 错误提示 |
|---|---|---|---|
| 手机号 | 11 位数字 | `/^1[3-9]\d{9}$/` | "请输入正确的手机号" |
| 密码 | 6-20 字符 | 非空 | "密码至少 6 位" |
| 验证码 | 6 位数字 | 60s 倒计时 | "验证码已发送" / "请稍后再试" |
| 昵称（注册） | 2-12 字 | 非空 | "请输入昵称" |
| 位置（注册） | 文本 | 选填 | — |

#### 交互逻辑
1. 进入页时检测 `storage.token`，**若已登录直接跳 P2**
2. 点登录 → 校验 → 调 `POST /api/auth/login` → 成功存 `token + userInfo` → `wx.switchTab` 到 P2
3. 失败显示 toast：账号不存在 / 密码错误 / 网络异常
4. 验证码登录：点"获取验证码"先校验手机号格式，调 `POST /api/auth/sms`，按钮变为 `60s` 倒计时
5. 协议未勾选 → 登录按钮置灰，点击 toast "请先同意用户协议"

---

### 3.2 P2 首页"我有需求"

#### 页面元素（自上而下）
1. **顶部用户卡片**（绿色 #4CAF50 渐变背景）
   - 左：头像 + 昵称 + 位置（如 "湖南省娄底市XX村"）
   - 右：「+ 发布需求」白色圆角按钮
2. **三宫格统计卡**：待接单 N / 今日订单 N / 总服务次数 N
3. **订单列表**（自己发布的，倒序）
   - 卡片字段：订单号 · 取件 → 送达 · 距离 + 预计时间 · 费用（红色加粗）· 状态徽章 · 加急标签 · 备注摘要 · 现金到付标签
4. **空态**：暂无订单 → "去发布第一个需求"按钮

#### 按钮交互

| 元素 | 行为 |
|---|---|
| 顶部「+ 发布需求」 | `wx.navigateTo` → P6 |
| 订单卡片 | `navigateTo` → P7（携带 order_id） |
| 下拉刷新 | `onPullDownRefresh` → 重新拉列表 |
| 上拉加载 | 分页 page+1，每页 10 条 |
| 状态徽章颜色 | pending=橙 / ongoing=蓝 / completed=绿 / cancelled=灰 |

#### 接口
- `GET /api/orders?user_id=xxx&role=publisher&page=1&size=10`

### 3.3 P3 我来帮忙

#### 页面元素
1. 顶部筛选栏：`全部` / `附近 5km` / `附近 10km` / `加急`
2. 排序按钮：默认按距离近 → 远，可切换"价高优先"
3. 订单列表（**不显示自己发布的单**）
   - 字段：取件 → 送达 · 距离 · 费用 · 加急标签 · 发布者昵称 + 评分
   - 右下角「接单」按钮（橙色）

#### 交互逻辑
1. 进入页面调用 `wx.getLocation` 获取经纬度（首次需弹授权）
2. 拒绝授权 → 提示"无法显示距离，请到设置开启" + 兜底默认排序按时间倒序
3. 点「接单」→ 弹 modal："确认接单后请尽快与发布者联系，是否接单？"
4. 确认 → `PUT /api/orders/{id}/accept` → 成功 toast "接单成功" → 跳 P7

#### 业务规则
- 一个跑腿员同时进行中订单 ≤ 3（防止抢单不送）
- 自己的订单不出现在列表里（后端过滤 `runner_id != user_id AND publisher_id != user_id`）

### 3.4 P4 订单页

#### 页面元素
- 顶部 2 个内 Tab：`我的订单` / `我来帮忙`
- 状态筛选 chips：`全部` / `待接单` / `进行中` / `已完成` / `已取消`
- 搜索框：支持订单号、地址、备注的关键词模糊搜索（前端本地过滤）
- 订单列表（同首页卡片样式）

#### 交互
- Tab 切换 → 切换查询参数 `role=publisher | runner`
- 状态筛选 → `?status=pending`
- 关键词搜索 → 输入即过滤（防抖 300ms）
- 点击卡片 → P7

### 3.5 P5 个人中心

#### 页面元素（卡片化布局）
1. **用户卡**：头像、昵称、手机号、位置、注册天数
2. **收益卡**：累计收益 / 本月收益 / 待提现 → 点「申请提现」跳 P8
3. **功能列表**：

   - 编辑资料（昵称、位置、头像）
   - 修改密码
   - 用户协议
   - 隐私政策
   - 关于我们
   - **退出登录**（红字）

#### 交互
- 退出登录 → 弹 modal 二次确认 → 清 storage → `reLaunch` 到 P1
- 编辑资料 → 弹底部弹层，保存调 `PUT /api/users/me`

### 3.6 P6 发布需求

#### 表单字段（必填项以 ※ 标记）

| 字段 | 控件 | 校验 |
|---|---|---|
| ※ 取件地址 | 文本 + 地图选点 | 非空，长度 ≤ 50 |
| ※ 送达地址 | 文本 + 地图选点 | 非空，长度 ≤ 50 |
| ※ 物品描述 | 文本框 | 非空，2-100 字 |
| ※ 配送费用 | 数字（0.01 步进） | > 0，≤ 500 |
| 期望送达时间 | 时间选择器 | 默认"尽快"，可选 1h/2h/今日内/明日 |
| 是否加急 | switch | 默认关，加急有红色标签 |
| 支付方式 | radio | 现金到付 / 在线支付（v1 默认现金） |
| 备注 | 文本框 | ≤ 200 字 |

#### 按钮
- 底部固定「发布需求」绿色大按钮（高度 88rpx）
- 点击 → 校验 → `POST /api/orders` → 成功 toast → 1s 后 `navigateBack`

#### 距离/费用提示
- 选完两个地址，自动调用高德 API 计算距离 + 预计时间，显示在表单上方
- 距离 > 20km 时弹提示："距离较远，建议适当提高费用"

### 3.7 P7 订单详情（最复杂的一页）

#### 页面布局（分卡片展示）

```
┌────────────────────────────────┐
│  状态卡（顶部，颜色随状态变）       │
│  「等待帮帮员接单」+ 倒计时          │
├────────────────────────────────┤
│  地址卡                          │
│  📍 取件：xxx                     │
│  🏁 送达：xxx                     │
│  距离 xx km · 预计 xx 分钟          │
├────────────────────────────────┤
│  订单信息卡                        │
│  物品 / 费用 / 时间 / 备注 / 加急    │
├────────────────────────────────┤
│  帮帮员卡（已接单后显示）            │
│  头像 / 昵称 / 评分 / 电话按钮       │
├────────────────────────────────┤
│  操作区（按角色+状态展示不同按钮）   │
└────────────────────────────────┘
```

#### 操作按钮矩阵（核心规则）

| 当前状态 | 我是发布者看到 | 我是跑腿员看到 |
|---|---|---|
| pending（待接单） | 取消订单（红） / 编辑（暂未做） | 接单（绿） |
| ongoing（进行中） | 联系帮帮员 / 取消（要原因） | 确认送达（绿） / 联系发布者 |
| completed（已完成） | 去评价帮帮员 | 查看评价（如有） |
| reviewed（已评价） | 查看评价 | 查看评价 |
| cancelled（已取消） | 显示取消原因 | 显示取消原因 |

#### 关键交互弹窗

1. **取消订单弹窗**
   - 标题"取消订单"
   - 必填 textarea：取消原因（≥ 5 字）
   - 「取消」「确认取消」两按钮
   - 点确认 → `PUT /api/orders/{id}/cancel` → 状态变 cancelled → 列表刷新

2. **送达确认弹窗**
   - 文案："已将物品送达？确认后无法撤回"
   - 点确认 → `PUT /api/orders/{id}/complete` → 状态变 completed
   - 后端自动给跑腿员加收益记录

3. **评价弹窗**（点"去评价"打开）
   - 5 颗大星 tap 选择
   - textarea 评价内容（选填，≤ 200 字）
   - 提交 → `POST /api/reviews` → 状态变 reviewed
   - 后端自动重算跑腿员平均评分 + 好评率（≥4 星算好评）

4. **打电话**
   - 点电话按钮 → `wx.makePhoneCall`
   - 隐藏号：v1 直接展示，v2 接入隐私号中转

### 3.8 P8 申请提现

#### 页面元素
- 可提现金额（大字）
- 提现方式：微信零钱（默认）
- 输入框：提现金额（≤ 可提现金额，≥ 1 元）
- 实名校验提示（首次提现需输入真实姓名 + 身份证号后 4 位）
- 「立即提现」按钮

#### 规则
- 单笔上限 200 元，单日上限 500 元（防风险）
- 实名信息一旦绑定不可改
- 提现成功后状态："审核中" → 1-3 工作日到账（v1 人工审核）

## 4. 异常与边界情况处理

### 4.1 网络与请求异常

| 场景 | 处理方式 |
|---|---|
| 请求超时（>10s） | toast "网络较慢，请稍后重试" |
| 后端 5xx | toast "服务异常，工程师正在抢修" |
| 后端 401 token 失效 | 清 storage → 跳 P1 + toast "登录已过期" |
| 后端 403 权限不足 | toast "无操作权限"（如取消别人的订单） |
| 后端 404 资源不存在 | 详情页显示空态："订单已被删除或不存在" |
| 离线状态 | 操作时统一拦截 toast "请检查网络连接" |

### 4.2 空数据状态

| 页面 | 空态文案 + 操作 |
|---|---|
| 首页订单列表 | "您还没有发布过需求" + 「立即发布」按钮 |
| 我来帮忙 | "附近暂无待接订单，过会儿再来看看" |
| 订单页（按状态） | "暂无 [状态名] 的订单" |
| 我的评价 | "暂无评价，赶紧去帮帮邻居吧" |
| 收益记录 | "暂无收益，去接单赚零花钱" + 跳 P3 按钮 |

### 4.3 权限与限制

| 场景 | 处理 |
|---|---|
| 拒绝定位授权 | 距离不显示，订单按时间倒序，顶部黄条提示"开启定位查看附近订单" + 跳设置 |
| 拒绝相册/相机（v2 上传凭证） | toast "需要授权才能上传图片" + 引导跳设置 |
| 未登录访问 P2-P9 | App.vue 全局拦截 → reLaunch P1 |
| 跑腿员同时 ≥3 个 ongoing | 接单按钮置灰，文案"请先完成手头的订单" |
| 1 小时内取消 ≥3 单 | 暂停发布权限 24h（防恶意刷单）+ toast 说明 |
| 单笔订单费用 > 500 | 表单校验阻止提交 + toast "费用过高，请联系客服" |

### 4.4 数据边界

| 数据 | 边界规则 |
|---|---|
| 长地址（>30 字） | 列表卡片单行省略号，详情页完整展示 |
| 长备注（>100 字） | 列表显示前 30 字 + ...，详情页完整 |
| 长昵称（>8 字） | 列表/详情页固定 8 字省略 |
| 0/负数费用 | 前端阻止 + 后端二次校验 |
| 重复提交 | 提交按钮 loading 期间置灰，2s 内同接口幂等去重 |
| 订单号格式 | `YYMMDD + 6位随机`，全局唯一 |

### 4.5 兼容性兜底

| 场景 | 处理 |
|---|---|
| 老人字号过大 | 全局 `pages.json` 设置最大字号兼容，关键卡片用 rpx 不用 px |
| 弱网首次加载 | 骨架屏 0.5s + 加载失败时显示「点击重试」 |
| 服务降级 | 高德 API 异常 → 距离/时间显示 "—"，不阻塞下单 |

## 5. 埋点与适配要求

### 5.1 埋点事件清单（v1 必埋）

> 命名规范：`模块_动作_对象`，全部小写下划线。统一上报到 `POST /api/track`。

| 事件名 | 触发时机 | 关键参数 |
|---|---|---|
| `app_launch` | 小程序启动 | `scene`（启动场景值）、`version` |
| `login_success` | 登录成功 | `method`（password/sms）、`user_id` |
| `login_fail` | 登录失败 | `reason`、`phone` |
| `home_view` | 进入首页 | `user_id` |
| `publish_start` | 进入发布页 | — |
| `publish_submit` | 点击发布按钮 | `fee`、`distance`、`is_urgent` |
| `publish_success` | 发布成功 | `order_id` |
| `runner_view` | 进入"我来帮忙" | `filter`（5km/10km/urgent） |
| `order_accept` | 跑腿员点接单 | `order_id`、`fee` |
| `order_complete` | 跑腿员点送达 | `order_id`、`duration_min` |
| `order_cancel` | 取消订单 | `order_id`、`role`、`reason` |
| `review_submit` | 提交评价 | `order_id`、`rating` |
| `withdraw_apply` | 提交提现 | `amount` |
| `error_occur` | 任何接口报错 | `api`、`code`、`msg` |

> **公共参数**（每条都带）：`user_id`、`timestamp`、`platform`（mp-weixin）、`version`、`network`（wifi/4g/5g/none）

### 5.2 关键转化漏斗（运营关注）

```
启动 → 登录 → 首页 → 发布页 → 提交 → 发布成功
启动 → 登录 → 我来帮忙 → 接单 → 送达 → 评价
```

### 5.3 适配要求

#### 设备适配
| 项目 | 要求 |
|---|---|
| 屏幕尺寸 | 主测 iPhone 12/13/14（375×812），兼容 iPhone SE（375×667）和大屏安卓（414+） |
| 单位 | 一律使用 rpx（750rpx = 屏宽），禁止用 px 写布局 |
| 安全区 | 底部 TabBar 自动加 `env(safe-area-inset-bottom)`，刘海屏头部用 `wx.getSystemInfoSync().statusBarHeight` |
| 横屏 | 不支持，`pages.json` 全局禁用 |

#### 系统适配
| 项目 | 要求 |
|---|---|
| 微信版本 | 基础库 ≥ 2.19.0 |
| iOS | iOS 12 及以上 |
| Android | Android 7 及以上 |
| 网络 | 弱网/无网兜底已在 4.1 |

#### 字体与无障碍（适老化）
- 关键文字最小 28rpx（约 14px），按钮内文字 ≥ 32rpx
- 按钮高度 ≥ 88rpx，老人手指好点
- 主要操作按钮用纯文字，不依赖图标识别
- 颜色对比度 ≥ 4.5:1（WCAG AA）

#### 性能要求
| 指标 | 标准 |
|---|---|
| 首屏渲染 | < 1.5s（4G） |
| 接口响应 | P95 < 1s |
| 包体积 | 主包 < 1.5MB（小程序限 2MB） |
| 内存 | 长时间使用不崩溃，列表大于 50 条用虚拟滚动 |

### 5.4 合规与安全
- 用户隐私协议、用户协议必须可点击查看（小程序提审硬要求）
- 所有接口走 HTTPS
- 敏感字段（密码、身份证）前端 sha256 后传输，后端 bcrypt 存储
- 手机号显示中间 4 位打码：`138****8004`
- 高德地图 Key 限制 referer/包名
