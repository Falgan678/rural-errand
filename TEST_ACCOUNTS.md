## 登录方式

### 方式1：用户名 + 密码登录
1. 打开登录页面
2. 选择"密码登录"
3. 输入用户名（如：测试用户1 或 falgan）
4. 输入对应密码（测试用户1-3的密码是123456，falgan的密码是@#Gfl3068690）
5. 勾选协议
6. 点击"登录"

### 方式2：手机号 + 密码登录
1. 打开登录页面
2. 选择"密码登录"
3. 输入手机号（如：13800138001 或 13800138004）
4. 输入对应密码
5. 勾选协议
6. 点击"登录"

### 方式3：手机号 + 验证码登录（开发模式）
1. 打开登录页面
2. 选择"验证码登录"
3. 输入手机号（如：13800138001）
4. 点击"获取验证码"
5. 查看服务器控制台或访问 `/api/debug/sms-codes` 获取验证码
6. 输入验证码
7. 勾选协议
8. 点击"登录"

## 快速测试流程

### 测试发布订单
1. 使用账号1登录（测试用户1）
2. 点击"发布跑腿需求"
3. 填写订单信息并提交

### 测试接单
1. 使用账号2登录（测试用户2）
2. 切换到"我当跑腿"
3. 查看待接订单
4. 点击"接单"按钮

### 测试订单管理
1. 登录任意账号
2. 点击底部"订单"按钮
3. 查看不同状态的订单
4. 切换"我有需求"和"我当跑腿"查看不同角色的订单

## 重置测试数据

如需重置测试账号数据，可以执行以下SQL：

```sql
-- 删除测试账号的订单
DELETE FROM orders WHERE user_id IN (
  SELECT id FROM users WHERE phone LIKE '138001380%'
);

-- 重置测试账号数据
UPDATE users 
SET total_orders = 0, 
    total_earnings = 0, 
    month_earnings = 0, 
    pending_earnings = 0,
    rating = 5.0,
    good_rate = 100.0
WHERE phone LIKE '138001380%';
```
